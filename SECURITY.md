# 数据与安全策略

## 数据分类

- **禁止进入 Git**：原始导出、完整脱敏业务导出、姓名、手机号、证件号、密钥、Webhook、真实群地址、脱敏映射表。
- **默认允许**：字段定义、业务规则、完全合成数据、无人员身份的报表模板。
- **条件允许**：经过自动扫描、人工复核和最小化抽样的脱敏业务样本；提交说明必须写明复核人、复核日期和用途。

“脱敏后的真实数据”不等于无风险数据。校区、班型、时间、备注等字段组合仍可能造成重识别，因此默认只在 `data/local/` 中处理。

## 输入防护

- 拒绝含 Excel 公式、外部工作簿链接或以 `= + - @` 开头的文本单元格。
- 拒绝关键标识重复、比例越界、月份无效和容量为负的数据。
- 不使用宏，不执行输入文件中的代码，不访问工作簿内的外部 URL。

## 输出防护

- 消息载荷仅生成草稿，并包含 `requires_human_confirmation: true`。
- 不在输出中回显人员身份字段。
- 所有结论携带数据日期、配置版本、输入哈希和证据键。

## 事件处理

发现敏感数据已进入 Git 时，停止推送并立即通知仓库负责人。仅删除工作区文件不足以清除 Git 历史；必须按仓库安全流程重写历史、轮换可能泄露的凭据，并重新审查所有克隆副本。
