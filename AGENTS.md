# 项目协作规则

## 沟通

- 使用中文回复。
- 用户概念不准确时，先给出更准确术语及其影响，再继续工作。
- 设计方案从目标、约束、事实和可验证结果出发，不从工具或框架出发。

## 实施

- 只实现当前明确要求，不预建第二个 Skill、Plugin、线上推送或排课系统。
- 数值计算必须在确定性脚本中完成；模型只能解释结果，不能补写缺失数字。
- 业务规则、字段映射和决策阈值进入版本化配置，不散落在提示词和代码常量中。
- 修改保持手术式：不重构无关代码，不清理用户已有内容。

## 数据安全

- `data/local/` 和 `outputs/` 不得提交。
- Git 中只允许合成数据，或经过自动扫描和人工复核的最小化脱敏样本。
- 不记录姓名、手机号、证件号、密钥、Webhook、真实群地址或可逆映射表。
- 输入 Excel 出现公式、外部链接或疑似公式注入文本时必须阻断。

## 验收

- 正常算例与人工基准逐项一致。
- 对缺列、重复记录、越界率、日期错位、容量冲突和恶意单元格做对抗性测试。
- 关键输入失败时，只输出校验报告，不得给出招聘或扩租结论。
- 交付前执行：

```bash
python3 -m unittest discover -s tests -v
python3 tools/scan_sensitive_data.py .
python3 tools/validate_skill.py .agents/skills/xdf-plan-campus-capacity
```
