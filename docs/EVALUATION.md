# 评测与对抗性验收

## 评测单位

每个用例包含输入工作簿、配置版本、任务提示、预期输出和可验证断言。评测必须在干净上下文中运行，不向执行者泄露预期答案。

## 校区产能正常用例

- 续费前后人数变化正确。
- 往期达成率正确作用于新增学员。
- 搭班教师需求、共享教室类型和三种情景正确。
- 最小班容、目标班容和最大班容共同约束班级数，边界人数能给出正确开班状态。
- 招聘与扩租建议的提前月份、持续期和证据键正确。
- 相同输入、配置和数据日期重复运行得到相同 JSON 与 Markdown。

## 校区产能对抗用例

- 缺少必填工作表、表头被改名或关键单元格为空。
- 学员标识重复，续费率大于 1 或小于 0。
- 班容顺序错误、负容量、月份格式错误。
- 同一教师或教室在同一月份和时段重复占用。
- 输入含公式、外部链接或以 `= + - @` 开头的文本。
- 数据日期超过允许时效。
- 历史目标为零或缺失，导致招生达成率不可计算。

## 教师课表规整正常用例

- 暑假按轮次、秋季按星期与时段形成教师占档矩阵，有效周三课程能动态增列。
- 未达开班线、已开未满和满班/超售的字体颜色与配置一致。
- 班号和班级名冲突时以班号为主并提示；NL1/NL2 基础与拔高规则正确。
- 容量汇总只包含总容量、当前数量和差额，差额公式可复算且允许负数。
- 计划新班按约定格式合并计算，当前数量为 0。
- 多个源校区可通过重复参数或配置管理单元合并，摘要保留实际源校区清单。
- 周课与暑假连续课混合时，按唯一密集片段识别轮次和主时段。
- 多个人工模板生成数据源新增、模板遗留、教师/时间变更三向清单，范围外编码不误报。
- 多名授课教师时，唯一且匹配的主带课老师入表，工号不进入输出。
- `rule_first` 保留数据源位置，`template_first` 可复现回传模板的删班、换教师和换时段。
- 受限本地规则覆盖能改变确定性结果，恢复后回到初始结果，初始配置字节不变。

## 教师课表规整对抗用例

- 错误校区名、缺列、重复班号、非法人数、公式、外部链接和疑似公式注入。
- 非标准课次、缺教师、无法识别的时段、非法计划新班格式和陈旧班号。
- 同教师同时段存在唯一高人数候选或并列高人数候选。
- 人工模板中伪装为班量/容量计算的未授权公式。
- 多个连续课片段落入不同轮次或时段、非法校区管理单元、重复源校区配置。
- 多名授课教师但主带课老师缺失、多值或不在授课教师列表中。
- 模板同班号多位置、跨季移动、非法时段，以及 `template_first` 试图恢复源内不合格班。
- 自然语言补丁试图修改源字段、安全或冲突策略；初始配置哈希变化后试图静默复用旧覆盖。
- 校验问题与排除记录分表；空冲突表给出明确无冲突结论；容量总览与时间下钻总额一致。

## 判分

- 机械断言由测试脚本判定，必须给出实际值与期望值。
- 管理建议由业务负责人做盲审，重点看正确性、可执行性、证据充分性和是否越权。
- 关键安全断言失败即整次评测失败，不用平均分掩盖。

## 上线门槛

1. 自动测试全通过。
2. Skill 结构校验通过。
3. 敏感信息扫描通过。
4. 至少三个历史周期回测通过。
5. 一线管理者书面确认口径与输出模板。
