# 校区产能规划项目规格

## 目标与使用者

首期服务儿童素养智学部一线管理者，把系统导出的 Excel 转换为可复算的教师招聘和教室扩租决策依据。系统降低数据整理时间，但不替代管理者审批。

## 问题定义

管理决策依赖四个连续问题：未来有多少学员、需要多少班、峰值时段需要多少教师和教室、现有供给何时出现持续缺口。任何后续建议都必须能回到这四步的输入与公式。

## 首期范围

- 按校区、月份、峰值时段和班型输出保守、基准、进取三种情景。
- 使用在读、续费率、招生目标、往期达成、开课时间、班容、搭班、教师和教室供给。
- 输出清洗后的标准表、预测明细、缺口、建议、Markdown 报告和连接器中立 JSON。

## 非目标

- 不做具体排课、教师绩效评价、招生归因或财务预算。
- 不自动执行招聘、扩租或消息发送。
- 不在首期引入机器学习、OR-Tools、数据库或实时业务系统连接。

## 决策边界

只有下列条件同时满足时才生成管理建议：关键表完整、字段与类型通过校验、历史目标大于零、续费率有效、资源主键无冲突、输入不含公式或外部链接。否则输出问题清单并停止建议。

## 成功标准

- 合成基准算例的关键数值完全一致。
- 同一输入与配置产生相同的 JSON 和管理报告。
- 每条建议包含首次缺口月份、峰值缺口、持续月份、提前期、情景和证据键。
- 三个历史周期回测通过后，业务方确认结果可用于例会讨论。
- 记录原人工流程后，数据处理时间至少下降 50%。

## 术语

- **产能**：指定月份和峰值时段内，在既定班型、班容和搭班规则下可承载的教学需求。
- **需求容量**：预测班级数换算出的教师并发和教室并发需求。
- **供给容量**：已知可用教师并发上限或可用教室数量。
- **缺口**：需求容量减供给容量后大于零的部分。
- **搭班**：一个班同时需要多名教师的配置，以“每班教师数”进入确定性计算。
