# 秋季纳新名额测算：候选 Skill 发现记录

## 合并状态

2026-07-16 已将本地项目《秋季纳新名额测算》完整复制到 `data/local/autumn-enrollment-quota/source-project/`。该目录受 Git 忽略，保留真实工作簿、中间数据、预览和一次性脚本，不进入 GitHub。

本文档只保留可复用的业务记忆和风险边界，已排除姓名、班级编码、微信临时路径和真实校区结果。

## 概念校正

源项目当前计算的是“已有秋季班人数 + 后续轮次续班预估人数”，并将班级映射到教师和时段。更准确的名称是“秋季续班承接与预排产能测算”。

真正的“可纳新名额”应计算为：

```text
可纳新名额 = 可售总容量 - 已续班占用 - 预计续班占用 - 预留名额
```

在可售总容量、教室/教师硬约束或预留口径未提供时，不得把已测得的预计人数直接解释为可纳新名额。

## 独立触发条件

当管理者需要把夏季班级、秋季已续班、秋季预排和续班率组合成“哪些班应放入哪个时段、预计占用多少人和多少班”时，使用该管理动作。

它与 `xdf-plan-campus-capacity` 的区别是：

- 本动作面向一个招生季的班级续转和可售名额。
- `xdf-plan-campus-capacity` 面向未来 12 个月的教师与教室资源缺口。
- 本动作可作为产能规划的需求侧输入，但不应与长周期产能预测共用一套触发条件。

## 已恢复的输入合同

### 班级明细工作簿

- 工作表：`暑假明细`、`秋季明细`。
- 关键字段：上课地点、班级编码、班级名称、当前占名额人数、最大人数、开结课日期、上课时间、授课教师、教室、备注、年级、产品品类。

### 校区运营工作簿

- 工作表：`秋季预排`、`秋季排课带名额版`。
- 逻辑维度：项目、教师、星期/时点、班型标签、轮次、已占用单元格与未排槽位。

源表名、列名和枚举值必须进入版本化配置；实际封装 Skill 时不得保留固定行号、列范围和本机绝对路径。

## 已恢复的确定性规则

1. 按目标校区过滤班级明细。
2. 已有秋季班以秋季明细的最新当前人数刷新。
3. 从夏季明细筛选需要向秋季承接的后续轮次班级，原班人数必须大于零。
4. 班级编码解析为课型、子课型、年级和难度；无法解析时必须阻断，不得猜测。
5. 候选预排槽位先按项目和班型等硬条件匹配，再用教师、轮次和时点作为排序证据。
6. 一个班级只能映射到一个槽位，同一槽位的冲突必须显式报告。
7. 待续班预估人数为 `ceil(原班人数 × 对应课型续班率)`。
8. 已有秋季班按当前人数计入，不再乘续班率。
9. 未确定教师或时段的班级保留在待排清单，不计入已排产能。
10. 汇总按课型、子课型、年级、难度分组，人数和班级数必须能回溯到映射明细。

## 源运行中的已确认参数

- STEAM 续班率：92%。
- 思维续班率：75%。
- 博学续班率：85%。
- 预估人数：向上取整。
- 特定非标准系列和定金班：源运行中不纳入，封装前必须转成可配置的排除规则并由业务负责人复核。
- 低年级脑力思维的难度归类：源运行默认归入 A，封装前需业务复核。

上述参数是一次业务运行中的已确认记忆，不是永久常量。未经新周期复核不得直接套用。

## 不得泛化的一次性知识

- 教师姓名校正、新教师占位符和姓名别名。
- 针对某个班级编码的固定单元格覆盖。
- 用 Excel 日期序列值直接识别轮次。
- 只针对某个校区名字的字符串过滤。
- 固定的工作表数据范围和输出 UUID 目录。

这些内容是当次人工修正或运行环境信息，只能作为评测“低置信需人工确认”的反例，不能进入通用规则。

## 对抗性审查结论

- 源运行的核心输出表“秋季排课带名额版”、“产能计算”和“续班测算明细”未记录到常见公式错误。
- 整个输出工作簿的其他原有工作表仍有大量 `#N/A`。因此只能认定核心新增表通过局部验收，不能宣称整个工作簿无错。
- 源运行依赖人工覆盖与姓名别名，还不具备“换一份数据也能安全重跑”的 Skill 级稳定性。
- 该源项目的数据与预览含真实运营信息，只能保留在 Git 忽略目录。

## 封装为第二个 Skill 前的门槛

1. 业务负责人确认“续班承接”与“可纳新名额”的区别。
2. 补齐班级可售容量、预留名额和教师/教室约束口径。
3. 把续班率、排除系列、轮次日期、编码解析和匹配权重转入版本化配置。
4. 建立不含姓名、班级编码和真实人数的合成评测数据。
5. 对缺表/缺列、重复班级、无法解析编码、无候选槽位、一槽多班、同分候选、越界续班率、未排班级和公式错误建立回归测试。
6. 低置信匹配只输出候选和证据，不自动写入正式排课表。

达到以上门槛后，再创建独立的仓库级 Skill；当前只作为候选管理动作和本地参考实现。
