# 数据合同

## 总则

输入为 `.xlsx`，第一行必须是表头。源表名和表头可在 `config/capacity-planning.json` 中映射，脚本不猜测相似列。月份统一使用 `YYYY-MM` 或 Excel 日期；比例使用 0 到 1 的数值。

## 元数据

`元数据` 使用“字段/值”两列，必须提供：

- `data_as_of`：数据快照日期，格式 `YYYY-MM-DD`。
- `source_system`：来源系统或导出流程名称。
- `template_version`：输入模板版本。

## 逻辑表

### 在读学员

一行代表一名脱敏后的在读学员。`学员标识` 必须是不可逆标识且全表唯一；状态仅接受“在读”或“停课”。续费月份决定何时应用续费率。

### 续费率

以“校区 + 班型 + 续费月份”为唯一键。基准续费率必须位于 `[0, 1]`。

### 招生计划

以“校区 + 班型 + 开课月份 + 峰值时段”为业务键。本期目标、往期目标和往期实际均不得为负；往期目标必须大于零；开课月份不得早于招生月份，否则不能生成决策建议。

### 班型规则

每个班型一行。满足 `0 < 最小班容 <= 目标班容 <= 最大班容`，每班教师数和教室数为正数，并指定教室类型。

### 教师供给

一行代表某教师在一个月份、峰值时段和班型下的可用容量。同一教师不得在同一月份和时段重复出现，否则视为并发冲突。

### 教室供给

一行代表某教室在一个月份、峰值时段下是否可用。同一教室不得在同一月份和时段重复出现，否则视为并发冲突。

## 计算口径

- 续费前的在读学员按 1 人计；到达续费月份后按对应情景续费率计，结果是期望人数，可为小数。
- 招生达成率为 `往期实际人数 / 往期目标人数`；新增学员为 `本期目标人数 × 招生达成率 × 情景系数`，从开课月份开始累计。
- 低于最小班容时计划班级数为 0 并标记待处理；其余情况先满足最大班容，再在最小班容允许的范围内选择最接近目标班容的班级数。若上下限无可行解，则按最大班容计算资源并标记末班低于下限。
- 教师需求为 `计划班级数 × 每班教师数`。
- 教室需求按“校区 + 月份 + 峰值时段 + 教室类型”汇总 `计划班级数 × 每班教室数`。
- 缺口为 `max(0, 需求 - 供给)`。

## 安全约束

所有输入单元格必须是静态值。公式、外部链接、疑似公式注入文本、手机号、证件号和密钥不得进入评测数据或 Git。
