# 数据合同

## 输入

输入必须是有效 `.xlsx`，所有源单元格必须为静态值。工作表名和字段映射由 `config/teacher-schedule.json` 明确指定。

CLI 必填：

- 校区范围二选一：重复提供一个或多个 `--campus` 做精确匹配，或用 `--campus-group` 选择配置中的管理单元。
- `--data-as-of`：教务系统导出快照日期，格式 `YYYY-MM-DD`。

## 规范字段

每个季节表映射为以下字段：

| 规范字段 | 必填 | 含义 |
|---|---|---|
| subject | 是 | 源系统科目名称，用于与班号前缀交叉验证 |
| campus | 是 | 完整校区名称 |
| class_code | 是 | 班级唯一编码 |
| class_name | 是 | 班级名称，用于范围和班型交叉验证 |
| current_students | 是 | 当前在读/占名额人数，非负整数 |
| source_max_students | 是 | 源系统最大人数，仅作配置规则交叉验证 |
| start_date | 是 | 开课日期 |
| schedule_text | 是 | 上课时间文本 |
| teacher | 是 | 教师姓名，可带工号；输出前移除工号 |
| primary_teacher | 否 | 源表“主带课老师”；仅在授课教师为多人时用于唯一选择 |
| class_sessions | 是 | 班级总课次 |
| open_status | 否 | 是否放班；配置中的关闭值不进入结果 |
| source_grade | 否 | 源系统年级，用于交叉验证 |
| round | 否 | 寒暑假轮次；缺失时可由配置日期范围判断 |
| remarks | 否 | 备注，用于排除取消、定金、复习、补课班 |

源表中的“班级级别”字段不参与判定；班型由班级编码解析，班级名称和年级字段只作交叉验证。

## 唯一性与安全

- `class_code` 在全部已配置季节表中必须唯一；重复即阻断。
- 缺少必填工作表、必填列或关键值时阻断。
- 公式、外部链接、以 `= + - @` 开头的文本阻断。
- 输入只在本机或授权环境处理；真实源表和输出不得进入 Git。

## 人工模板

人工模板必须来自本技能输出，可重复提供暑假、秋季独立文件。只自动读取标准正式班单元格和以 `新班` 开头的计划班单元格。对有效正式班执行三向对比：数据源有而模板无、模板有而数据源无、同班号教师或时段不同。

- `rule_first`：差异进入待确认，不应用正式班删改。
- `template_first`：在已提供模板的季节内，模板决定有效正式班是否保留以及教师/时段；缺教师、非法时段、重复位置、跨季移动和源内不合格班不能应用。

## 本地规则覆盖

`--rules-overlay` 只读取 `manage_schedule_rules.py` 生成的 JSON，包含初始配置 SHA-256、允许路径覆盖和本地历史。哈希不一致时阻断。初始配置是恢复基准，不由 AI 直接编辑；覆盖和真实数据一样必须位于 Git 忽略路径。
