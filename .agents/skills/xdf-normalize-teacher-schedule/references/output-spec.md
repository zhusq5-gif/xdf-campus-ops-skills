# 输出与视觉规范

## 工作簿

- `运行摘要`：状态、数据日期、管理口径、纳入源校区、源记录数、排除数、输入哈希、配置版本、规则来源、模板优先级、问题数量、模板差异数量和人工确认要求。
- `校验问题`：阻断、警告和待确认；不混入按规则正常排除的记录。
- `排除记录`：S2/S3、定金/取消/复习/补课班、关闭班及不支持科目；说明列必须展示实际命中的排除词或关闭状态。
- `暑假课表`：教师 × 轮次时段矩阵。
- `秋季课表`：教师 × 星期时段矩阵；默认周四至周日，遇到周一至周三有效班级时动态增加。
- `容量总览`：按季节、科目、年级、难度/班型汇总，总容量、当前数量、差额。
- `容量汇总`：按时间和班级来源继续下钻，总容量、当前数量、差额。
- `容量明细`：每个有效班级一行。
- `排课冲突`：冲突候选、人数和临时选择结果。
- `待确认`：非标准课次、无法识别的年级/时段、同人数冲突、正式班人工移动和其他模板异常。
- `模板对比`：数据源新增、模板遗留、源内但未通过统计规则、同班号教师/时间变化、模板已应用变更和模板移除源班。
- `假设与版本`：合并后配置展开结果，包含初始/本地覆盖模式和初始配置哈希。

排课冲突为空时必须显示“本次未发现教师时段冲突”，不得只留下空表头。“班型编码”统一使用更准确的列名“难度/班型”。

## 配色

时间背景：

- 周中：`#F4F7FB`
- 周六：`#EAF3FF`
- 周日：`#FFF3E6`
- 第一轮：`#EAF3FF`
- 第二轮：`#F3EEFF`
- 第三轮：`#FFF3E6`

状态字体：

- 未达到开班线：`#1D4ED8`
- 已开班未满：`#157A3D`
- 满班或超售：`#B42318`
- 普通文字：`#1F2937`

主表头使用 `#2F5597` 白字；科目列使用 `#DCE6F1`；教师列使用 `#F8FAFC`；边框使用 `#CBD5E1`。状态字体在全部时间背景上的对比度不得低于 4.5:1。

## 单元格格式

正式班：

```text
18人
ST3AQ270001
```

计划新班保留约定简写，例如 `新班ST3A`。班量必须等于该教师行有效占档单元格数，不能写成平均人数。
