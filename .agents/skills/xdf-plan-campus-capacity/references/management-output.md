# 管理输出标准

## 解释顺序

1. 先说明数据日期、配置版本和运行状态。
2. 再给基准情景的首次缺口、峰值和持续期。
3. 对比保守和进取情景，指出结论是否稳定。
4. 列出建议动作、最迟启动月份和证据键。
5. 最后列出数据风险、未覆盖约束和需要人工确认的事项。

## 合格建议

合格建议必须同时包含：资源类型、校区、班型或教室类型、峰值时段、首次缺口月份、峰值缺口、连续月数、提前期、建议截止月份、证据键。

不要把“预测缺口”写成已经发生的事实，不要把“建议招聘 2 个容量单位”擅自解释成具体编制或劳动合同数量；业务方确认教师工时和岗位口径后再落到招聘计划。

## 消息载荷

`message-payload.json` 是连接器中立草稿。部署 Agent 可以把它转换为飞书或钉钉消息，但发送前必须显示预览并取得人工确认。
