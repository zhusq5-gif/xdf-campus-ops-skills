# 数据合同使用说明

在首次接入或源表变化时，先阅读仓库根目录 `docs/DATA_CONTRACT.md`，再修改 `config/capacity-planning.json` 中的工作表和列映射。

## 映射原则

- 把源表列显式映射到规范字段，不按相似名称猜测。
- 保持规范字段名稳定；源系统变更只修改映射。
- 月份只接受 Excel 日期、`YYYY-MM` 或 `YYYY-MM-DD`。
- 比例使用 0 到 1 的数值，不接受百分号文本。
- 人员和教室只使用不可逆标识，不在输出回显真实身份。

## 接入检查

1. 确认 `元数据` 中有 `data_as_of`、`source_system` 和 `template_version`。
2. 确认六张逻辑表都能映射到规范字段。
3. 先在合成或最小化脱敏样本上运行。
4. 查看 `校验问题`，阻断项为零后才允许业务回测。
5. 源表或模板变化后重跑全部自动测试和历史回测。
